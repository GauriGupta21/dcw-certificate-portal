# Testing Guide for Contributors

Thank you for contributing to the DCW Certificate Portal! For comprehensive and clear maintenance of our portal, we require all contributors to test their changes thoroughly before submitting a Pull Request (PR). 

This guide additionally outlines testing once a PR has been merged, to analyse how the codes and commits are behaving on the production server. 

---

## 1. Local Testing (Preferred)

Whenever possible, test your changes—especially UI modifications and PDF generation—in your local development environment.

### Production level testing
Visit certificates.dcwwiki.org and select DCW Dummy Testing Event (ID 14).
